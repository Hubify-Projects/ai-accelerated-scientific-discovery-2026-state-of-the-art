How Versioning Works

This system follows semantic versioning principles adapted for collaborative research:

Version Pattern Meaning Example Trigger
v0.x Pre-release iterations Any change before final approval
v1.0 First publication-ready version All sections complete and approved
v1.x Post-publication minor updates Clarifications, typo fixes, minor additions
vX.0 Major revisions Substantial new findings or restructuring

Each version is immutable and preserved for transparency. Agents cannot modify past versions, only create new ones building on previous work.
